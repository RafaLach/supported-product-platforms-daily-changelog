Introduces new inventory entities: Foundry IQ agents, knowledge base connections, and IQ query sessions. AISPM must discover and map these resources and their data flow to knowledge bases. Source: https://learn.microsoft.com/en-us/azure/foundry/agents/concepts/what-is-foundry-iq

- **[AISPM] [HIGH] Azure Foundry Control Plane**: New centralized management layer for agents, models, and tools in Azure AI Foundry. Includes agentic workflow evaluation and GenAI application monitoring. Creates a new governance surface — AISPM should inventory Control Plane-managed resources and track agent-model-tool relationships. Source: https://learn.microsoft.com/en-us/azure/foundry/control-plane/overview

- **[AISPM] [HIGH] Amazon Bedrock — 9 new models & 3 new providers**: Moonshot AI (Kimi K2.5), MiniMax (M2.1), and Zhipu (GLM 4.7/Flash) are entirely new providers. Plus Claude 4.6 variants, DeepSeek V3.2, Qwen3 Coder Next, and Nova 2. AISPM model inventory must be expanded to discover and classify these models and their deployments. Source: https://docs.aws.amazon.com/bedrock/latest/userguide/models-supported.html

- **[AISPM] [HIGH] Amazon Bedrock OpenAI-compatible Projects API**: New API surface in Mantle inference engine provides OpenAI API compatibility. Extends the existing AgentCore API surface — AISPM should track calls to this endpoint for agent activity visibility. Source: https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/runtime-execute-command.html

- **[AISPM] [MEDIUM] Google Vertex AI — Gemini 3.1 Pro & GLM 4.7 GA**: New models available in Vertex AI Model Garden. AISPM model inventory for Vertex AI should include these models and track their deployments. Source: https://docs.cloud.google.com/vertex-ai/generative-ai/docs/models/gemini/3-1-pro

- **[AIDR] [MEDIUM] Bedrock AgentCore OpenInference protocol**: New observability protocol for agent tracing. AIDR can leverage OpenInference traces as a detection data source for monitoring agent behavior. Source: https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/observability.html

### AIRT — New Attack Surfaces

- **[AIRT] [HIGH] OpenAI MCP Apps — tool execution attack surface**: MCP apps connecting external tools to ChatGPT introduce prompt injection vectors through tool descriptions and results. AIRT should test MCP app tool chains for injection vulnerabilities and data exfiltration via tool responses. Source: https://developers.openai.com/apps-sdk/concepts/mcp-server

- **[AIRT] [HIGH] Bedrock A2A protocol — inter-agent attack surface**: Agent-to-Agent communication creates new prompt injection vectors where a compromised agent could manipulate other agents via A2A messages. AIRT should test A2A chains for cross-agent prompt injection and privilege escalation. Source: https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/runtime-a2a-protocol-contract.html

- **[AIRT] [MEDIUM] Azure Foundry IQ — knowledge base poisoning surface**: Foundry IQ connects knowledge bases to agents, creating a data poisoning vector. AIRT should test whether malicious content in knowledge bases can manipulate Foundry IQ agent responses. Source: https://learn.microsoft.com/en-us/azure/foundry/agents/concepts/what-is-foundry-iq
