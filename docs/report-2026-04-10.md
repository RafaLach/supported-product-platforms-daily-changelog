## Platform Details

### Amazon Bedrock AgentCore

- **Agent Registry service launched** (Apr 9): New centralized catalog for discovering and managing agents, MCP servers, tools, and skills within AgentCore. Provides governed workflows for publishing, reviewing, and approving resources with hybrid semantic and keyword search. Registry is now listed as a core AgentCore service alongside Runtime, Memory, Gateway, Identity, Code Interpreter, Browser, Observability, Evaluations, and Policy. New API action: `SearchRegistryRecords`. Can be accessed via console, APIs, or as an MCP server directly from IDEs. Available in preview in 5 AWS regions. Source: https://aws.amazon.com/about-aws/whats-new/2026/04/aws-agent-registry-in-agentcore-preview/

### ChatGPT Enterprise (OpenAI)

- **`web_search_call.results` added to ResponseIncludable** (Apr 8, Python SDK v2.31.0): Web search call results can now be included as a response includable type, expanding the data returned in API responses. Source: https://developers.openai.com/api/docs/changelog
- **`phase` field added to Conversations message type** (Apr 8, SDK v2.31.0 / v6.34.0): New field on conversation messages across both Python and Node SDKs. Source: https://developers.openai.com/api/docs/changelog
- **Short-lived token authentication** (Apr 8, SDK v2.31.0 / v6.34.0): New client authentication mechanism using short-lived tokens, available in both Python and Node SDKs. Source: https://developers.openai.com/apps-sdk/build/auth
- **Conversations API response compaction** (community discussion): New capability for managing conversation length limits via response compaction. Source: https://community.openai.com

### Salesforce AgentForce

- **Agent "Topics" renamed to "Subagents"** (Apr 2026): Beginning April 2026, Salesforce has renamed agent "topics" to "subagents" across all AgentForce and Einstein GenAI documentation. Salesforce states this is a terminology/concept realignment with no functional changes. Source: https://www.salesforce.com/blog/agentforce-guide/

### Cursor

- **Bugbot MCP Support** (Apr 8): Bugbot code review agent now has access to MCP servers, giving it full tool-use context during code reviews rather than just static analysis. Source: https://cursor.com/changelog/04-08-26
- **Bugbot Learned Rules** (Apr 8): Bugbot now self-improves by turning user feedback signals into learned rules that shape future code reviews. New entity type: learned rules. 110K+ repos enabled, 44K+ rules generated. Source: https://cursor.com/blog/bugbot-learning
- **Bugbot "Fix All" action** (Apr 8): Users can apply all Bugbot-suggested fixes in a single operation. Source: https://cursor.com/changelog/04-08-26

### Dia Browser

- **Cross-device Sync** (Apr 9, v1.26.0): Dia launched browser data synchronization across multiple devices using 6-character pairing codes. Data is end-to-end encrypted. Includes recovery kit creation. This is a major new capability for a previously local-first browser. Source: https://diabrowser.com/release-notes

## New Coverage Required

- **[AISPM] [HIGH] AWS Bedrock AgentCore Agent Registry**: New centralized catalog service for discovering and managing agents, MCP servers, tools, and skills. Introduces new inventory entities: registry records, agent listings, MCP server listings, tool listings, and skill listings with governed publish/review/approve workflows. New API action `SearchRegistryRecords` provides hybrid semantic and keyword search across registry contents. The registry is also accessible as an MCP server from IDEs. AISPM must discover, inventory, and map all registry resources and their relationships. Source: https://aws.amazon.com/about-aws/whats-new/2026/04/aws-agent-registry-in-agentcore-preview/

- **[AIDR] [MEDIUM] OpenAI web_search_call.results ResponseIncludable**: The OpenAI Responses API now supports `web_search_call.results` as an includable type, meaning web search results can be embedded directly in API responses. This creates a new data flow surface — external web content flowing through the API into enterprise applications. AIDR should monitor these results for data exfiltration and content injection risks. Source: https://developers.openai.com/api/docs/changelog

- **[AIDR] [MEDIUM] OpenAI Short-lived Token Authentication**: New authentication mechanism using short-lived tokens for OpenAI API clients, replacing long-lived API keys in cloud-managed environments. Changes how sessions are established and authenticated, affecting identity tracking and session monitoring. AIDR must recognize this auth pattern to maintain audit trail coverage. Source: https://developers.openai.com/apps-sdk/build/auth

- **[AISPM] [MEDIUM] Salesforce AgentForce Subagents entity model change**: Salesforce renamed agent "topics" to "subagents" across AgentForce and Einstein GenAI. While described as a terminology change, this signals a conceptual shift in how Salesforce models agent composition — subagents are now positioned as specialized agents with specific expertise within a parent agent. AISPM inventory must update entity type mappings from "topics" to "subagents" to maintain accurate discovery and governance of Salesforce agent structures. Source: https://www.salesforce.com/blog/agentforce-guide/

- **[ENDPOINT] [MEDIUM] Cursor Bugbot MCP Server Access**: Cursor's Bugbot code review agent now connects to MCP servers for full tool-use context during reviews. This expands the attack surface — a compromised or malicious MCP server could influence code review outcomes, and Bugbot now has access to tools and data beyond the code itself. Endpoint monitoring should track Bugbot's MCP connections and tool invocations. Source: https://cursor.com/changelog/04-08-26

- **[AIRT] [MEDIUM] Cursor Bugbot Learned Rules**: Bugbot now self-improves by converting feedback into persistent learned rules that shape future code reviews (110K+ repos, 44K+ rules). This creates a new prompt injection / rule poisoning vector — adversarial feedback could train Bugbot to ignore security issues or approve malicious patterns. AIRT should evaluate rule poisoning as an attack surface. Source: https://cursor.com/blog/bugbot-learning
