## New Coverage Required

- **[AISPM] [HIGH] Copilot Studio Work IQ Integration**: New Work IQ service integration (preview) enables Copilot Studio agents to access real-time work insights from Microsoft 365 files, emails, meetings, and chats. Introduces new entity types: Work IQ-connected agents, work insight data connections, and M365 context bindings. AISPM must discover agents with Work IQ enabled and map their data access patterns — these agents gain broad read access to organizational work data across M365 services, creating potential data leakage vectors. Source: https://learn.microsoft.com/en-us/microsoft-copilot-studio/use-work-iq

- **[AISPM] [HIGH] Copilot Studio Computer Use Expansion**: Computer Use agents in Copilot Studio gained expanded capabilities: new model support (Claude Sonnet 4.5 beta), built-in credentials management, Cloud PC pooling, and enhanced audit logging with session replay. Computer Use agents can control mouse/keyboard and interact with desktop applications. AISPM must inventory Computer Use agents, their model bindings, credential stores, and Cloud PC allocations. The session replay audit logging is a new data artifact that should be tracked. Source: https://www.microsoft.com/en-us/microsoft-copilot/blog/copilot-studio/computer-use-is-now-in-public-preview-in-microsoft-copilot-studio/

- **[AIDR] [HIGH] Copilot Studio Computer Use Audit Logging and Session Replay**: Computer Use agents now generate enhanced audit logs with full session replay capability. This is a new activity data surface — every screen interaction, credential usage, and agent action during a Computer Use session is recorded. AIDR should ingest these audit logs and session replays to detect anomalous agent behavior, unauthorized credential usage, or data exfiltration through desktop application interactions. Source: https://www.microsoft.com/en-us/microsoft-copilot/blog/copilot-studio/computer-use-is-now-in-public-preview-in-microsoft-copilot-studio/

- **[AIRT] [HIGH] Copilot Studio Computer Use with Claude Models**: Computer Use agents can now use Claude Sonnet 4.5 for decision-making and control desktop applications with mouse/keyboard. This is a significant attack surface: prompt injection through on-screen content could manipulate agent actions, and credential-enabled agents operating on Cloud PCs introduce privilege escalation risks. AIRT should test Computer Use agents for visual prompt injection, credential abuse, and unauthorized application access scenarios. Source: https://www.microsoft.com/en-us/microsoft-copilot/blog/copilot-studio/computer-use-is-now-in-public-preview-in-microsoft-copilot-studio/

- **[AISPM] [MEDIUM] Copilot Studio Multi-Model Prompt Builder**: Copilot Studio prompt builder now supports Claude Opus 4.6 and Claude Sonnet 4.5 model selection alongside existing models. Introduces per-prompt content moderation sensitivity configuration (low/high) for multiple content categories. AISPM model inventory must be updated to track which prompts use which models, and the content moderation configuration per prompt represents a new governance surface — misconfigured moderation could allow harmful content generation. Source: https://learn.microsoft.com/en-us/microsoft-copilot-studio/prompt-model-settings

- **[ENDPOINT] [MEDIUM] Copilot Studio VS Code Extension (GA)**: The Copilot Studio extension for Visual Studio Code reached general availability. Developers can now build and manage Copilot Studio agents directly from VS Code. Endpoint monitoring should track this extension as a local AI development surface that connects to the Copilot Studio platform. Source: https://learn.microsoft.com/en-us/microsoft-copilot-studio/visual-studio-code-extension-overview

- **[ENDPOINT] [MEDIUM] OpenAI Codex Sub-Agent Architecture and Plugin System**: Codex v0.117.0 introduced a first-class plugin system with startup sync, TUI browsing, and path-based sub-agent addressing (`/root/agent_a`). Codex v0.118.0 added `codex exec` command for prompt-plus-stdin workflows and dynamic bearer token management for custom model providers. The sub-agent architecture means Codex sessions can now spawn and coordinate multiple autonomous agents. Endpoint monitoring should track Codex plugin configurations, sub-agent spawning, and the new network egress enforcement on Windows. Source: https://developers.openai.com/codex/plugins
