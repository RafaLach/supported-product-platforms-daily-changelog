## New Coverage Required

- **[AISPM] [HIGH] Anthropic Claude Opus 4.7**: New frontier Anthropic model released April 16, 2026 with stronger coding, agent, vision, and multi-step task capabilities. Already rolling out to Vertex AI Model Garden (April 14) and will appear on Amazon Bedrock and Microsoft Foundry. Ships with automated cybersecurity safeguards that detect and block high-risk cyber requests, and Anthropic specifically trained it to "differentially reduce" offensive-cyber capabilities. AISPM must add Claude Opus 4.7 to model inventory across all integrated platforms and track its deployment scope, since agent-use characteristics differ from prior Opus versions. Source: https://www.anthropic.com/news/claude-opus-4-7

- **[AISPM] [HIGH] Anthropic Claude Design (Anthropic Labs)**: New Claude product (April 17, 2026) powered by Claude Opus 4.7 that generates designs, prototypes, slides, and one-pagers — expanding Claude's surface from chat/code into design-artifact creation. During onboarding, Claude reads your codebase and design files to build a team design system automatically; every project then uses those colors, typography, and components. Introduces new entity types for AISPM to track: Claude Design workspaces, team design systems (ingested from codebase), generated prototype/deck artifacts, and Canva/PPTX export sinks. Zenity needs to map how enterprise codebase and brand content flows into Claude Design. Available in research preview on Pro, Max, Team, and Enterprise. Source: https://www.anthropic.com/news/claude-design-anthropic-labs

- **[AIRT] [HIGH] OpenAI Codex expands beyond coding (in-app browser, computer use macOS, PR workflow, 90+ new plugins)**: Major April 16, 2026 Codex update dramatically expands attack surface. New capabilities include in-app browser (live web interaction from the agent), computer use for macOS ("seeing, clicking, and typing with its own cursor" across any native app, with multiple agents in parallel), GitHub PR workflow integration (direct diff review and comment), thread automations (scheduled autonomous follow-ups), artifact viewer (PDFs/spreadsheets/docs/slides), and 90+ new plugins including Atlassian Rovo, CircleCI, GitLab Issues, Microsoft tools. Each is a new vector for prompt injection, agent-driven data exfiltration, and unauthorized action. AIRT threat models must cover browser-in-agent, macOS computer use, and scheduled-task invocation. Source: https://developers.openai.com/codex/app/computer-use

- **[ENDPOINT] [HIGH] OpenAI Codex Marketplace Plugin Installation from arbitrary git URLs**: Codex CLI 0.121.0 (April 15, 2026) added support for installing plugin marketplaces from GitHub, git URLs, local directories, and direct marketplace.json URLs. This introduces a supply-chain surface: developers can install third-party Codex extensions that run locally with the agent's permissions — any repo can ship skills, tools, and MCP servers that the agent will then invoke. ENDPOINT monitoring should inventory installed marketplace plugins and flag unvetted sources. Source: https://developers.openai.com/codex/plugins

- **[AISPM] [MEDIUM] Google Vertex AI RAG Cross Corpus Retrieval (public preview)**: New April 17, 2026 feature enables agentic retrieval across multiple RAG corpora in a single query. Backed by two new APIs: `AsyncRetrieveContexts` (long-running) and `AskContexts` (synchronous answer generation). Uses an Orchestrator/Planning Agent that reads each corpus description to decide where to search — so corpus descriptions become a new security-relevant metadata field. AISPM must discover cross-corpus configurations, map which corpora an agent is authorized to aggregate, and flag overly permissive corpus descriptions. Currently us-central1 only. Source: https://docs.cloud.google.com/vertex-ai/generative-ai/docs/rag-engine/cross-corpus-retrieval

- **[AISPM] [MEDIUM] Google Vertex AI RAG Engine Serverless Mode**: New April 3, 2026 public-preview deployment mode for RAG Engine — fully managed Vector Search 2.0 collection provisioned automatically, no infrastructure configuration. Configured via `GetRagEngineConfig`/`UpdateRagEngineConfig` APIs at the project level. AISPM must discover Serverless-mode RAG corpora as a distinct inventory entity separate from Spanner-mode corpora, and track `UpdateRagEngineConfig` events as governance-relevant changes. Source: https://docs.cloud.google.com/vertex-ai/generative-ai/docs/rag-engine/serverless-mode

- **[AISPM] [MEDIUM] Google Vertex AI Gemma 4 26B A4B IT (experimental)**: New multimodal Gemma 4 MoE model (April 3, 2026) added to Vertex AI Model Garden. 26B total parameters but only 3.8B active per inference step; accepts text and image input. AISPM model inventory must track Gemma 4 deployments; multimodal image input expands the prompt-injection surface beyond text. Source: https://docs.cloud.google.com/vertex-ai/generative-ai/docs/maas/google/gemma-4-26b-a4b-it

- **[ENDPOINT] [HIGH] Windsurf 2.0 with Embedded Devin Cloud Agent**: Windsurf 2.0 (April 15, 2026) ships with Devin cloud agent embedded directly inside the Windsurf IDE, included with every self-serve plan (Pro, Max, Teams). Adds an Agent Command Center (Kanban board for local + cloud agents) and a new "Spaces" organizational unit grouping agent sessions, PRs, files, and context by task. Devin runs on its own VM with desktop, browser, and computer-use capabilities — it can continue working after the user closes their laptop. ENDPOINT monitoring must now treat Windsurf-resident Devin invocations as a distinct agent identity that can spawn cloud VMs and execute multi-step tasks independently. Source: https://windsurf.com/blog/windsurf-2-0

- **[ENDPOINT] [MEDIUM] Cursor Canvases and Cursor 3.1 Tiled Layout**: Cursor 3.1 (April 13, 2026) adds Tiled Layout for running/managing several agents in parallel panes; Canvases (April 15) are a new durable artifact type rendered alongside chat — agents can generate interactive React interfaces containing charts, tables, diff views, and custom logic via user-written Skills. Canvas Skills (e.g. Docs Canvas Skill on Cursor Marketplace) define what kind of canvas an agent can generate, creating a new inventory entity: Canvas Skills as a form of agent-granted capability. ENDPOINT must track per-session concurrent agents, inventory canvas artifacts, and flag custom Canvas Skill installations. Source: https://cursor.com/docs/agent/tools/canvas

- **[AIDR] [MEDIUM] Perplexity `/v1/agent` canonical endpoint + `/v1/models`**: Perplexity promoted `/v1/agent` to the canonical Agent API endpoint (March 2026) — replacing the previous `/v1/responses` path (still accepted as an OpenAI-compatible alias). New `GET /v1/models` endpoint (April 2026) lists all available Agent API models in OpenAI-compatible format. Added third-party models: GPT-5.4, NVIDIA Nemotron, Claude Sonnet 4.6, Gemini 3.1 Pro Preview — so a single Perplexity call can now invoke an OpenAI, Anthropic, Google, or NVIDIA model with built-in `web_search` and `fetch_url` tools. AIDR activity monitoring must recognize `/v1/agent`, attribute cross-provider model usage correctly, and inventory Perplexity as a new multi-provider inference gateway. Source: https://docs.perplexity.ai/docs/agent-api/quickstart
