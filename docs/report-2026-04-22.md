## New Coverage Required

- **[ENDPOINT] [HIGH] Codex CLI 0.122.0 deny-read filesystem policies + isolated sandbox enforcement**: Codex CLI 0.122.0 (April 20, 2026) adds **deny-read glob policies** to the filesystem permission model — a new per-path control that prevents the local agent from reading files matching a pattern, even when the CLI otherwise has read access to the project. This is a governance-relevant control surface for endpoint monitoring: administrators can now express "never let this agent read files matching `**/.env`, `**/secrets/**`" etc., and **isolated sandbox enforcement** is now a distinct runtime mode. ENDPOINT must inventory which Codex CLI sessions run with deny-read policies configured (and which do not), treat absence or overly-permissive glob lists as a finding, and correlate sandbox vs. non-sandbox sessions with the blast radius of agent-initiated writes/exec. Source: https://developers.openai.com/codex/changelog

- **[AIRT] [MEDIUM] Codex CLI 0.122.0 `/side` conversations and queued slash commands**: The `/side` feature lets the user (and in scripted flows, other tools) inject a concurrent conversation while the primary agent task is still running. Together with **queued slash commands**, this creates a new class of mid-session state manipulation: a `/side` injection can alter context, change models, or run tools while the main loop is paused between steps. AIRT red-teaming should cover (a) `/side` used to bypass permission prompts that the primary session had already consented to, (b) queued slash commands dropped in via automation that fire after a /clear or context change, and (c) race conditions between the queued command list and active tool execution. Tool discovery and image generation are now on by default, which widens the baseline capability surface of every new session. Source: https://developers.openai.com/codex/changelog

- **[AISPM] [MEDIUM] Salesforce AgentForce "subagents" rename (topic → subagent)**: AgentForce renamed every "agent topic" to **subagent** across the Developer Guide, Agentforce DX, Agent Script, and API references as of April 2026. Although Salesforce states "no changes to functionality," the rename touches metadata API field names, Apex types, and documentation URLs — so any Zenity AgentForce adapter that references the literal `topic` / `agentTopic` entity names needs to accept both forms during the transition, and the AISPM inventory schema should normalize on `subagent` as the canonical term while keeping a backward alias. New subagents surfaced through this rename should still be inventoried as the same underlying capability, but the relationship modeling (parent agent → child subagent) is now a first-class concept in the product UI, which AISPM should reflect. Source: https://developer.salesforce.com/docs/ai/agentforce/guide
