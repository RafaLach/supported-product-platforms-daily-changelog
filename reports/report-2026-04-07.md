## Platform Details

### Microsoft Copilot Studio

The What's New page (updated 2026-04-07) expanded the March 2026 section significantly. The following entries were added since the last scan:

- **Agent evaluations GA**: Agent evaluations moved from preview to general availability. Validates agent performance using customizable test sets across diverse scenarios. Source: https://learn.microsoft.com/en-us/microsoft-copilot-studio/analytics-agent-evaluation-intro
- **Multi-turn conversation tests**: New capability to create multi-turn conversation tests to evaluate agents across realistic dialog flows instead of single-turn interactions. Source: https://learn.microsoft.com/en-us/microsoft-copilot-studio/analytics-agent-evaluation-multi-turn
- **ChatGPT-5 GA globally**: ChatGPT-5 is now generally available globally (excluding GCC) for use in production agents. Source: https://learn.microsoft.com/en-us/microsoft-copilot-studio/authoring-select-agent-model
- **Claude Sonnet 4.5, Claude Sonnet 4.6, Claude Opus GA globally**: Third-party Anthropic models now generally available globally (excluding GCC) for production agents, enabling fine-grained control over reasoning depth, quality, latency, and cost. Source: https://learn.microsoft.com/en-us/microsoft-copilot-studio/authoring-select-agent-model
- **Agent node for agent flows**: New capability to call a Copilot Studio agent from an agent flow, enabling agent-to-agent orchestration. Source: https://learn.microsoft.com/en-us/microsoft-copilot-studio/agent-node-workflow
- **Bing Custom Search as knowledge source**: Agents can now use Bing Custom Search to ground responses in a curated, scoped web index using a Custom Configuration ID. Source: https://learn.microsoft.com/en-us/microsoft-copilot-studio/knowledge-bing-custom-search
- **Post-call action topics for voice agents**: Voice-enabled agents can now trigger backend actions automatically after a call ends, based on conversation outcome. Source: https://learn.microsoft.com/en-us/microsoft-copilot-studio/voice-post-call-actions
- **Prompt assistant in Prompt builder**: GPT model-powered suggestions to draft prompts faster. Source: https://learn.microsoft.com/en-us/microsoft-copilot-studio/prompt-assistant
- **Reprompt messages with randomization**: Configure reprompt messages in Question nodes with randomization support for voice scenarios and entity recognition failures. Source: https://learn.microsoft.com/en-us/microsoft-copilot-studio/voice-random-message-selection
- **User questions and reactions analytics**: View and filter detailed lists of user questions and reactions from agent conversations; download to CSV. Source: https://learn.microsoft.com/en-us/microsoft-copilot-studio/analytics-question-reaction-lists
- **Session transcripts with CSAT values**: Downloaded session transcripts now include session-level customer satisfaction scores. Source: https://learn.microsoft.com/en-us/microsoft-copilot-studio/analytics-transcripts-studio

## New Coverage Required

### Microsoft Copilot Studio (Primary)

- **[AISPM] [MEDIUM] Copilot Studio Agent Node (Agent-to-Agent Orchestration)**: New agent node allows calling a Copilot Studio agent from an agent flow, enabling multi-agent orchestration chains. Introduces new entity relationships: agent flow → agent invocations. AISPM must map these agent-to-agent call chains to detect toxic combinations such as overprivileged agents invoking other agents or unmonitored delegation paths. Source: https://learn.microsoft.com/en-us/microsoft-copilot-studio/agent-node-workflow

- **[AISPM] [MEDIUM] Copilot Studio Bing Custom Search Knowledge Source**: Agents can now connect to Bing Custom Search as a knowledge source via Custom Configuration ID. Introduces new knowledge connection type: curated web index. AISPM must discover and inventory these connections and monitor for data exposure through externally-sourced web content flowing into agent responses. Source: https://learn.microsoft.com/en-us/microsoft-copilot-studio/knowledge-bing-custom-search

- **[AISPM] [MEDIUM] Copilot Studio Voice Post-Call Actions**: Voice-enabled agents can now trigger automated backend actions after call completion. Introduces new entity: post-call action topics with automatic execution triggers. AISPM must track these automated action configurations to detect unmonitored backend operations triggered by voice agent conversations. Source: https://learn.microsoft.com/en-us/microsoft-copilot-studio/voice-post-call-actions

- **[AISPM] [MEDIUM] Third-Party Models GA in Copilot Studio (ChatGPT-5, Claude Opus/Sonnet)**: ChatGPT-5 and multiple Anthropic Claude models (Sonnet 4.5, Sonnet 4.6, Opus) are now generally available globally for production agents. AISPM must track which external models production agents use, as model choice affects data processing boundaries — some models may require data processing outside the organization's geographic boundaries. Source: https://learn.microsoft.com/en-us/microsoft-copilot-studio/authoring-select-agent-model

- **[AIDR] [MEDIUM] Copilot Studio Multi-Turn Conversation Evaluation Tests**: New multi-turn conversation test capability for evaluating agents across realistic dialog flows. Agent evaluations are now GA. Introduces new data artifacts: test sets, evaluation results, and multi-turn conversation test configurations. AIDR should monitor evaluation data flows as they may contain sensitive conversation patterns. Source: https://learn.microsoft.com/en-us/microsoft-copilot-studio/analytics-agent-evaluation-multi-turn

### Google Vertex AI (Secondary)

- **[AISPM] [MEDIUM] Vertex AI RAG Engine Metadata Search**: New schema-based metadata filtering for RAG Engine corpora. Users can define metadata schemas, attach metadata to files, and filter contexts during retrieval. Extends RAG data flow surface with structured metadata that controls what knowledge is retrieved. AISPM should track metadata schema configurations as they influence agent knowledge access boundaries. Source: https://docs.cloud.google.com/vertex-ai/generative-ai/docs/rag-engine/use-metadata-search
