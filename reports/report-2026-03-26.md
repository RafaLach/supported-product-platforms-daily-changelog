## New Coverage Required

### AISPM — AI Security Posture Management

- **[AISPM] [HIGH] Bedrock AgentCore A2A Protocol Support**: Amazon Bedrock AgentCore now supports the Agent-to-Agent (A2A) protocol, enabling inter-agent communication across frameworks. Creates new entity types: A2A sessions, agent-to-agent message flows, and cross-framework agent connections. AISPM must discover and map A2A-enabled agents and their communication patterns to detect unauthorized inter-agent data flows. Source: https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/runtime-a2a.html

- **[AISPM] [HIGH] OpenAI GPT-5.4 Pro and Thinking Models**: OpenAI launched GPT-5.4 Pro and GPT-5.4 Thinking — new frontier model variants with 1M token context, Tool Search capability, and enhanced computer use. AISPM must update model inventory to discover enterprise deployments using these models, track which agents and applications reference them, and ensure governance policies cover the expanded reasoning capabilities of the Thinking variant. Source: https://openai.com/index/introducing-gpt-5-4/

- **[AISPM] [MEDIUM] Bedrock Server-Side Tools in Responses API**: Amazon Bedrock's Responses API now supports server-side tool execution via AgentCore Gateway, eliminating client-side orchestration. This shifts tool execution from visible client-side calls to opaque server-side execution. AISPM must discover and inventory server-side tool configurations to ensure tool access permissions are tracked. Source: https://aws.amazon.com/about-aws/whats-new/2026/02/amazon-bedrock-server-side-tool-execution-agentcore-gateway/

- **[AISPM] [MEDIUM] Bedrock Projects API (Mantle)**: OpenAI-compatible Projects API now supported in Bedrock's Mantle inference engine. Introduces new organizational entities (projects) that group models, deployments, and API keys with IAM-based access control and cost tagging. AISPM should discover project resources to map ownership and access boundaries. Source: https://docs.aws.amazon.com/bedrock/latest/userguide/projects.html

- **[AISPM] [MEDIUM] Bedrock AgentCore New Framework and Model Support**: AgentCore added Google ADK, OpenAI Agents SDK as supported frameworks and Google Gemini as a supported model. Expands the inventory of agent types that can run on AgentCore — AISPM must recognize agents built with these frameworks during discovery. Source: https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/using-any-agent-framework.html

- **[AISPM] [MEDIUM] Bedrock AgentCore Cedar Policy Language**: AgentCore Policy service now explicitly supports Cedar as a policy definition language alongside natural language. New entity type: Cedar policy documents attached to agents. AISPM should parse and inventory Cedar policies to assess agent permission boundaries. Source: https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/policy-understanding-cedar.html

- **[AISPM] [MEDIUM] Bedrock New Open-Weight Models**: Six new open-weight models added to Bedrock: DeepSeek V3.2, MiniMax M2.1, GLM 4.7, GLM 4.7 Flash, Kimi K2.5, Qwen3 Coder Next. AISPM model inventory must be updated to discover and track deployments using these models, especially for compliance with model governance policies. Source: https://aws.amazon.com/about-aws/whats-new/2026/02/amazon-bedrock-adds-support-six-open-weights-models/

- **[AISPM] [MEDIUM] OpenAI Open Responses API**: OpenAI's Responses API is now an open-source specification (backed by Hugging Face) enabling multi-provider interoperability. Enterprises may adopt this as a standard agentic interface across providers. AISPM should monitor for new Responses API-based agent deployments and cross-provider agent configurations. Source: https://community.openai.com/t/open-responses-for-the-open-source-community/1371770

### AIDR — AI Detection & Response

- **[AIDR] [MEDIUM] Bedrock AgentCore StopRuntimeSession API**: New `StopRuntimeSession` API action enables programmatic termination of agent runtime sessions. AIDR should monitor calls to this API as they may indicate incident response actions or unauthorized session terminations. Source: https://docs.aws.amazon.com/bedrock-agentcore/latest/APIReference/API_StopRuntimeSession.html

### AIRT — AI Red Teaming

- **[AIRT] [MEDIUM] OpenAI Computer Use Keys Field Expansion**: OpenAI SDKs (Python v2.30.0, Node v6.33.0) added a `keys` field to all computer action types (Click, DoubleClick, Drag, Move, Scroll), enabling keyboard input during computer use actions. Expands the attack surface for computer-use agents — AIRT should test for keystroke injection and keyboard-based privilege escalation in computer-use scenarios. Source: https://github.com/openai/openai-python/releases

- **[AIRT] [MEDIUM] OpenAI Playwright MCP in Codex**: OpenAI Codex now integrates Playwright MCP with approval workflows. Creates a new browser automation attack surface — AIRT should test for prompt injection via web content during Playwright-driven browsing sessions. Source: https://developers.openai.com/codex/mcp
