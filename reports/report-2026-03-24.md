## New Coverage Required

- **[AISPM] [HIGH] Microsoft Agent 365 (Early Access Preview)**: New agent platform entry added to M365 Copilot extensibility. Introduces a new agent entity type ("Microsoft Agent 365") in the M365 ecosystem that AISPM must discover and inventory alongside existing Copilot extensions, declarative agents, and API plugins. Microsoft Agent 365 is a new control plane for deploying, organizing, and governing agents at scale with integrated identity, compliance, and security from Entra and Defender. GA target: May 1, 2026. Source: https://learn.microsoft.com/en-us/microsoft-365-copilot/extensibility/whats-new

- **[AISPM] [HIGH] Copilot Studio Work IQ Tools Integration**: New service connecting Microsoft 365 Copilot and Copilot Studio agents to the Work IQ service for real-time work insights from M365 files, emails, meetings, and chats. Creates new data flow paths between agents and enterprise productivity data. AISPM must map agent-to-Work-IQ connections and AIDR must monitor data access patterns through this channel. Source: https://learn.microsoft.com/en-us/microsoft-copilot-studio/use-work-iq

- **[AISPM] [HIGH] Microsoft Foundry Rebrand with New Services (Foundry IQ, Control Plane, Local)**: Azure AI Foundry rebranded as "Microsoft Foundry" with three new service categories: Foundry IQ (knowledge-base-connected agents, evolved from Azure AI Search), Foundry Control Plane (tracing, evaluation, monitoring), and Foundry Local (local LLM execution on Windows/macOS/Android). Introduces new inventory entities for each service. AISPM must update its Azure AI resource discovery to cover the new Foundry namespace and entity types. Source: https://learn.microsoft.com/en-us/azure/foundry/what-is-foundry

- **[AIRT] [MEDIUM] Copilot Studio Computer Use Expansion**: Enhanced computer use capabilities with new model support (Claude Sonnet 4.5), built-in credentials management, Cloud PC pooling via Windows 365 for Agents, and enhanced audit logging with session replay. Expands the attack surface for agent-controlled desktop sessions — agents now have credential access and broader desktop control. AIRT should assess prompt injection risks in computer use scenarios and credential exposure vectors. Source: https://www.microsoft.com/en-us/microsoft-copilot/blog/copilot-studio/computer-using-agents-now-deliver-more-secure-ui-automation-at-scale/

- **[AISPM] [MEDIUM] Copilot Studio Third-Party Model Support (Claude Sonnet 4.5, Opus 4.6)**: Copilot Studio prompt builder now supports Claude models (Sonnet 4.5, Opus 4.6) alongside Microsoft models. Agents can use non-Microsoft LLMs, creating new model provider entities to track and new data flow paths to external model providers. AISPM must inventory which agents use which model providers. Source: https://www.microsoft.com/en-us/microsoft-copilot/blog/copilot-studio/available-today-claude-sonnet-4-5-in-microsoft-copilot-studio/

- **[AIDR] [MEDIUM] Copilot Studio Configurable Content Moderation Sensitivity**: Prompt builder now allows configuring content moderation sensitivity per individual prompt (Low/Moderate/High for hate, sexual, violence, self-harm categories). Creates risk of agents with weakened moderation settings. AIDR should detect and flag agents with lowered content moderation thresholds as a violation rule. Source: https://learn.microsoft.com/en-us/microsoft-copilot-studio/prompt-model-settings

- **[AISPM] [MEDIUM] Azure OpenAI GPT-Realtime-1.5 / GPT-Audio-1.5 Models**: New voice/audio models (gpt-realtime-1.5-2026-02-23 and gpt-audio-1.5-2026-02-23) released on Azure OpenAI for real-time voice-first applications with tool calling support. New model entities for AISPM inventory; voice-enabled agents introduce new interaction channels that expand the monitoring surface. Source: https://learn.microsoft.com/en-us/azure/foundry/openai/how-to/realtime-audio

- **[AISPM] [MEDIUM] Power Apps Code Apps and AI Experience Preview**: New "Code apps" development mode (now GA) added to Power Apps — enables code-first apps using React/Vue within Power Platform with enterprise governance. Plus a revamped AI copilot experience (preview) via vibe.powerapps.com. New app type entity for AISPM inventory — code apps may have different security characteristics than canvas/model-driven apps. Source: https://learn.microsoft.com/en-us/power-apps/developer/code-apps/overview

- **[AISPM] [MEDIUM] Amazon Bedrock Model and Provider Expansion**: Six new open-weight models (DeepSeek V3.2, MiniMax M2.1, GLM 4.7, GLM 4.7 Flash, Kimi K2.5, Qwen3 Coder Next) and three new providers (Moonshot AI, MiniMax, Zhipu) added to Bedrock, plus Amazon Nova 2. Powered by Project Mantle inference engine. Expands the model inventory AISPM must track and introduces providers from new geographies (China-origin models) with potential data sovereignty implications. Source: https://aws.amazon.com/about-aws/whats-new/2026/02/amazon-bedrock-adds-support-six-open-weights-models/

- **[AISPM] [MEDIUM] Amazon Bedrock Server-Side Tools via AgentCore Gateway**: Responses API now supports server-side tool execution through AgentCore Gateway integration without client-side orchestration. Customers specify an AgentCore Gateway ARN as a tool connector, and Bedrock auto-discovers tools, selects them during inference, and executes server-side — all within a single API call. AISPM must discover these tool configurations and AIDR must monitor server-side tool invocations as a new activity type. Source: https://aws.amazon.com/about-aws/whats-new/2026/02/amazon-bedrock-server-side-tool-execution-agentcore-gateway/

- **[AISPM] [MEDIUM] Amazon Bedrock OpenAI-Compatible Projects API (Mantle)**: New OpenAI-compatible Projects API surface in the Mantle inference engine (announced Feb 26, 2026). Projects provide logical boundaries for isolating workloads with IAM-based access control and cost tagging. AISPM must discover project entities alongside existing Bedrock resource structures. Source: https://aws.amazon.com/about-aws/whats-new/2026/03/amazon-bedrock-projects-api-mantle-inference-engine/

- **[AISPM] [HIGH] Salesforce Agentforce Voice (Spring '26)**: New voice-first agent capability delivering natural, human-like conversations aligned to company brand, with real-time CRM record updates, workflow triggering, and API calling — all with full auditability through Data 360. Natively connected to Salesforce with first-class support for Amazon Connect, Five9, Genesys, NiCE, and Vonage. AISPM must inventory voice-enabled Agentforce agents and their data access permissions. Source: https://www.salesforce.com/agentforce/voice/

- **[AISPM] [MEDIUM] Google Vertex AI Partner Model Evaluations**: Gen AI evaluation service now supports assessing third-party partner models (Anthropic, Llama) through console-based workflows via Model Garden. Extends the evaluation surface to non-Google models, which AISPM should track for model governance coverage. Source: https://docs.cloud.google.com/vertex-ai/docs/release-notes

- **[ENDPOINT] [MEDIUM] Windsurf Enterprise Cascade Hooks and MCP OAuth**: Cloud-configured cascade hooks for enterprise governance (v1.9552.21), auto-triggered OAuth for MCP connections, and new POST_CASCADE_RESPONSE_WITH_TRANSCRIPT hook type (v1.9566.9). Enterprise admin whitelist controls restrict which MCP servers developers can install using regex pattern matching. Source: https://windsurf.com/changelog
