## New Coverage Required

- **[AISPM] [HIGH] AWS Agent Registry in Bedrock AgentCore**: New centralized catalog service (launched April 9, 2026) for discovering and managing agents, MCP servers, tools, and skills within AgentCore. Features governed publishing/approval workflows and hybrid semantic+keyword search. Introduces new inventory entities: registry records, agent entries, MCP server entries, tool entries, skill entries, and approval workflows. AISPM must discover and catalog all resources published to the Registry, map their approval states, and flag ungoverned or unapproved agents. Source: https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/what-is-bedrock-agentcore.html

- **[AISPM] [MEDIUM] Google ADK Framework Support in Bedrock AgentCore Runtime**: Google Agent Development Kit added as a supported framework for AgentCore Runtime, joining CrewAI, LangGraph, LlamaIndex, OpenAI Agents SDK, and Strands Agents. Agents built with Google ADK can now be deployed and executed on AgentCore infrastructure. AISPM framework detection must be updated to identify and inventory Google ADK-based agents running in AgentCore. Source: https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/using-any-agent-framework.html

- **[AISPM] [MEDIUM] Anthropic Mythos Preview Model**: New "Mythos preview" model added to Anthropic's product lineup as a gated research preview for defensive cybersecurity work under Project Glasswing. Strikingly capable at computer security tasks — emerging capability not explicitly trained. AISPM should track Mythos model deployments and ensure governance policies cover this new model, particularly given its dual-use security capabilities. Source: https://www.anthropic.com/claude-mythos-preview-risk-report

- **[AIRT] [HIGH] Anthropic Mythos Preview — Exploit Generation Capability**: Mythos Preview demonstrates a step-change in autonomous exploit generation. Anthropic reports it developed working JavaScript engine exploits 181 times vs. Opus 4.6's 2, with register control achieved 29 more times. This dramatically expands the automated red teaming attack surface — AIRT must update threat models to account for Mythos-class exploit capability across all monitored platforms. Source: https://red.anthropic.com/2026/mythos-preview/

- **[AISPM] [MEDIUM] Claude Available via Microsoft Foundry**: Microsoft Foundry added as a new deployment platform for Claude models (Opus 4.6, Sonnet 4.6 in East US 2 and Sweden Central), alongside Amazon Bedrock and Google Cloud Vertex AI. This creates a new discovery surface — AISPM must now scan Microsoft Foundry for Claude model deployments in addition to existing platforms. Source: https://www.anthropic.com/news/claude-in-microsoft-foundry

- **[ENDPOINT] [MEDIUM] Claude Word Add-in**: Microsoft Word add-in now available for Claude, completing the Office integration surface (Excel, PowerPoint, Word, Slack). Enterprise users can draft, edit, and revise documents with Claude from a persistent sidebar, with edits appearing as tracked changes. Endpoint monitoring should track data flows between Word documents and Claude to detect sensitive document content exposure. Source: https://support.claude.com/en/articles/14465370-use-claude-for-word

- **[AISPM] [MEDIUM] Microsoft Graph Auth Protocol and Risk Detection Expansion**: New authentication protocol types (`nativeAuth`, `authenticationTransfer`, `samlOnBehalfOf`) added to Microsoft Graph API, expanding authentication pathways for Copilot and M365 integrations. Additionally, `microsoftRevokedSessions` added as a new `riskDetail` enumeration member, and `webCategory` resource type added for network compliance. AISPM should update permission and authentication monitoring to cover these new auth flows and risk signals. Source: https://developer.microsoft.com/graph/changelog/
